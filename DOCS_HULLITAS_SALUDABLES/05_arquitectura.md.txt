Arquitectura del Sistema:

Para el desarrollo de HUELLITAS SALUDABLES se propone una arquitectura de tipo cliente-servidor basada en tres capas, la cual facilita la organización del sistema, mejora su mantenimiento y permite su crecimiento a futuro.

Capa de Presentación (Frontend)

Corresponde a la interfaz gráfica con la que interactúan los usuarios del sistema. Su función principal es permitir la visualización de la información y la interacción con las funcionalidades del sistema.

Tecnologías sugeridas: HTML, CSS, JavaScript y frameworks como Angular o Vue.js.

Funciones principales:

Registro e inicio de sesión de usuarios
Visualización de perfiles
Búsqueda de servicios
Gestión de reservas
Capa de Lógica de Negocio (Backend)

Se encarga de procesar las solicitudes realizadas por los usuarios, aplicar las reglas del negocio y gestionar la comunicación entre la interfaz y la base de datos.

Tecnologías sugeridas: Node.js, Java o Python, mediante el uso de una API REST.

Funciones principales:

Validación de datos
Gestión de usuarios y roles
Administración de reservas
Implementación de seguridad y autenticación
Capa de Datos (Base de Datos)

Es la encargada de almacenar la información del sistema de manera organizada y segura.

Tecnología sugerida: PostgreSQL.

Funciones principales:

Almacenamiento de datos
Consultas y recuperación de información
Mantenimiento de la integridad de los datos
Flujo de funcionamiento del sistema
El usuario accede a la aplicación mediante la interfaz.
Realiza una solicitud (por ejemplo, buscar un cuidador).
La solicitud es enviada al servidor.
El servidor procesa la información y consulta la base de datos.
Se obtiene la respuesta y se envía nuevamente al usuario.

Adicionalmente, se puede implementar el patrón de arquitectura MVC (Modelo-Vista-Controlador) en el backend, con el fin de organizar el código y facilitar su mantenimiento.
